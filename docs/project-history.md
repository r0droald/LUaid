# LUaid Project History & Direction

**Source:** Hannah Paguila, Feb 24 2026

## Origin Story

1. Started as a transparency website for donations received, organizations involved, and relief distribution details.
2. Evolved into a tool for data collection, management, and visualization.
3. Hannah built a mobile-first PWA using free services (Google Sheets as DB, GitHub, WordPress CMS, free hosting).
4. Designed for offline-first use — caches data on-device, syncs when internet is available.
5. Secured domain `LUaid.org` and put up a placeholder site.
6. First transparency dashboard iteration: https://zippy-axolotl-a14a5e.netlify.app

> After initial planning and the first dashboard iteration, the project never got to an actual launch.

## Goals & Direction

- **Coordination tool** — automate relief effort processes
- **Transparency** — show donation and distribution data publicly
- **Knowledge sharing** — publish processes, learnings, and stories
- **Reusable template** — open-source tool for any future disaster, usable by anyone

## Website Vision

Serve as a working document and knowledge bank for sustainability and relief ops. Includes directories, journals, triage systems, and documentation of how self-organization works.

### Key Features

- Dashboard for updates and tracking
- Documentation of roles, responsibilities, and processes
- Open-source on GitHub with usage instructions
- Timeline, context, and gravity of the situation
- Signal Board / Bulletin Board (from Coi): lightweight internal reference for orgs to know "Who, What, Where" — avoids duplication, minimal volunteer burden

### Priority Pain Points

1. Deployment team communication
2. Offline data capture
3. Task creation / workflow (ticketing system)
4. Repacking logistics (auto-rank barangays by need)
5. Inventory and donation management
6. Donor acknowledgment page

## Sustainability

- **Costs:** Free-tier services (Google Sheets, Vercel hosting, free WordPress CMS). Only recurring cost is the `LUaid.org` domain (~PHP 600 for 2-3 years). Could fall back to a free subdomain if needed.
- **Tech maintenance:** Open-source GitHub project with prioritized tasks. Volunteer developers can contribute; collaboration guides being built.
- **Content:** Depends on sustained effort to publish processes, learnings, and stories.

## Relief Ops Context (as of Feb 2026)

- Schools: engaging district supervisors in San Juan for school needs
- Donation surplus: overflow of goods noted
- Deployment: bulk donations being sent to barangays and schools; reporting needed for donor accountability
